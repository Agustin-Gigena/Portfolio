## Anti-Patterns Verdict

**Would someone say "AI made this"? Marginally yes — the concentration of tells is just above threshold.**

### LLM Assessment

The site has a solid structural foundation and the DESIGN.md shows real thought about avoiding AI tells. But the code doesn't follow its own design system. The cluster that triggers the "AI made this" reaction:

- **Gradient text** on hero name (loudest tell)
- **Side-stripe 4px gradient borders** on every section title
- **Emoji icons** (8 instances: skill categories, contact links, theme toggle)
- **Hero-metric stats** (4-counter row — SaaS cliché)
- **Identical card grid** (4 project cards, same pattern)
- **Ghost-card hover** (1px border + 24px blur shadow on same card element)

Each is individually fixable. Together they create the unmistakable silhouette of a codex-generated template. The DESIGN.md explicitly bans 3 of these violations.

### Deterministic Scan

CLI detector found 14 findings: 2 warnings (overused-font, gradient-text), 1 warning (layout transition on `width`), 6 color advisories, 5 radius advisories. 

Browser inspection confirmed additional missed issues: no `:focus-visible` on any interactive element (critical a11y failure), no `aria-current` on nav links, JetBrains Mono loaded but unused (dead weight), and no custom form validation styling.

### Detector False Positives

- "Single font for everything" — JetBrains Mono IS loaded (just barely used)
- "#666 on #fff contrast issue" — passes WCAG AA at ~5.74:1
- "Ghost-card pattern" — technically border + 24px shadow coexist on hover, debatable

## Overall Impression

Solid structural foundation: semantic HTML, clean section layout, responsive at 768/480px, good body contrast. Content hierarchy is logical, and the timeline component is genuinely well-executed.

The problem is **the design system exists on paper but the code doesn't follow it**. The DESIGN.md bans gradient text, side-stripe borders, and emojis — the code uses all three. Closing this gap resolves most AI-tell issues at once.

## What's Working

1. **Timeline component.** Gradient vertical line + dot markers + amber dates + arrow-prefixed list items. Cohesive, intentional use of the three-accent palette. The amber dates are a legitimate application of the tertiary color.

2. **Tag hover transitions.** Gradient background, translateY(-2px), white text. Simple, satisfying, communicates interactivity without over-animation.

3. **Responsive breakpoints.** 768px and 480px thresholds are well-chosen. Grids collapse cleanly. Typography scales appropriately. Contact form stacks naturally.

## Priority Issues

### P0 — Gradient text on hero name
- **Where:** `.hero__name` (styles.css:328-332)
- **Why:** The loudest AI tell on the page. DESIGN.md explicitly bans it. Combined with the `< />` code glyph at 10rem and the radial glow, the hero has 3 decorative layers on top of real content.
- **Fix:** Replace with solid `color: var(--accent)` — straight Señal blue. Let weight and size carry emphasis.
- **Suggested:** `$impeccable quieter hero`

### P0 — Contact form has no backend
- **Where:** `index.html:255-268`, `ts/main.ts:147-164`
- **Why:** mailto: silently fails if no mail client is configured (common on mobile, corporate machines). Recruiters who fill out the form and get nothing back will not try again. Email address exposed to harvesters.
- **Fix:** Deploy Formspree, Web3Forms, or a serverless function. Add success/error UI states. At minimum, show an inline confirmation before opening mailto.
- **Suggested:** `$impeccable harden contact`

### P1 — No focus indicators on any interactive element
- **Where:** Global — `styles.css:730-733` removes `outline` on input focus but adds no `:focus-visible` anywhere
- **Why:** Keyboard-only navigation is invisible. WCAG AA failure (SC 2.4.7). Every interactive element (a, button, input) needs a visible focus ring.
- **Fix:** Add `a:focus-visible, button:focus-visible, input:focus-visible { outline: 2px solid var(--accent); outline-offset: 2px; }` globally. Re-evaluate `outline: none` on inputs.
- **Suggested:** `$impeccable audit`

### P1 — Side-stripe borders on every section title
- **Where:** `.section__title::before` (styles.css:239-248)
- **Why:** DESIGN.md absolute ban. Each title has double decoration — 4px left gradient stripe AND 64px bottom gradient bar. Two accent lines compete for attention.
- **Fix:** Remove `::before` entirely. Keep the bottom bar OR remove both and use font weight alone.
- **Suggested:** `$impeccable quieter section__title`

### P1 — Emoji icons throughout
- **Where:** 8 instances: skill categories (⚙️🎨🗄️🛠️), contact links (✉️💼🐙), theme toggle (🌙☀️)
- **Why:** Render inconsistently across OS and browser. Undercut the "precise technical" brand. DESIGN.md calls this out.
- **Fix:** Replace with inline SVGs. Skill icons → simple geometric outline icons. Contact → same SVG icons used in hero social links. Theme → SVG sun/moon.
- **Suggested:** `$impeccable colorize`

### P1 — No `prefers-reduced-motion` support
- **Where:** `.reveal` animations, counter animation, hover transforms (global)
- **Why:** WCAG AA requires respecting reduced motion (SC 2.3.3).
- **Fix:** Add `@media (prefers-reduced-motion: reduce)` setting `transition-duration: 0.01ms` / `animation: none` on reveals, counters, and transforms.
- **Suggested:** `$impeccable audit`

### P2 — Stats section is hero-metric template
- **Where:** `.stats` (index.html:60-77, styles.css:382-418)
- **Why:** 4-counter SaaS cliché. "10K+ hours" is dubious: 10,000 hours = 5.6 years full-time vs 4 years stated experience. If a recruiter does the math, trust erodes.
- **Fix:** Replace with a single strong testimonial, client logos, or remove entirely. If keeping, make numbers verifiable.
- **Suggested:** `$impeccable distill stats`

### P2 — Ghost-card hover pattern
- **Where:** `.project-card:hover`, `.stat:hover` (styles.css:620-624, 400-403)
- **Why:** `border: 1px solid` + `box-shadow: 0 8px 24px` on same element is the codex ghost-card tell. DESIGN.md flags this.
- **Fix:** Drop the border-color change on hover, keeping shadow alone. Or keep border change with blur ≤8px.
- **Suggested:** `$impeccable polish`

### P3 — text-wrap not applied
- **Where:** All headings and body text (global)
- **Why:** DESIGN.md requires `text-wrap: balance` on h1-h3 and `text-wrap: pretty` on long prose. Neither present.
- **Fix:** Add to hero name, section titles, body paragraphs.
- **Suggested:** `$impeccable typeset`

### P3 — JetBrains Mono loaded but unused
- **Where:** Google Fonts link in `<head>` (index.html:11)
- **Why:** 15KB+ font download for zero visible characters. No `<code>` or `<pre>` elements in the current DOM.
- **Fix:** Either remove the font load (saves bandwidth) or add technical code samples that justify it.
- **Suggested:** `$impeccable optimize`

### P3 — Navbar loses backdrop-filter on scroll
- **Where:** `ts/main.ts:53-59`
- **Why:** Scroll handler sets `background: var(--bg-primary)` at >50px scroll, overriding the `backdrop-filter: blur(12px)` and making the navbar opaque.
- **Fix:** Remove the scroll-based background override, or use a semi-transparent dark value that preserves the blur.
