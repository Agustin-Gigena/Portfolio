# i18n System Design

## Overview

Add internationalization (i18n) to the portfolio using data-* attributes on HTML elements. Supports Spanish (default) and English, with a modular system for adding more languages later.

## Approach

Use `data-es` and `data-en` attributes on each translatable HTML element. A TypeScript module reads the current language and swaps `textContent` dynamically. No URL changes, no external JSON files, no network requests.

## Architecture

### Files

| File | Purpose |
|------|---------|
| `ts/i18n.ts` | Language detection, toggle, text swapping |
| `index.html` | Add `data-es` / `data-en` to all translatable elements |
| `css/styles.css` | Style the language toggle button |

### Language Detection Flow

```
1. Check localStorage('lang')
2. If not set → check navigator.language
3. If not 'es' or 'en' → default to 'es'
4. Apply language to all elements
```

### Language Toggle

- Button in navbar next to theme toggle
- Shows current language label (ES / EN)
- Click cycles: ES → EN → ES
- Saves to localStorage

### HTML Structure

Each translatable element gets both attributes:

```html
<h1 data-es="Agustin Gigena" data-en="Agustin Gigena">Agustin Gigena</h1>
<p class="hero__title" data-es="Desarrollador .NET Full Stack" data-en=".NET Full Stack Developer">Desarrollador .NET Full Stack</p>
```

The default content in the HTML matches `data-es` (Spanish). When English is active, `textContent` is replaced with `data-en`.

### Elements to Translate

| Section | Elements |
|---------|----------|
| Hero | `.hero__title`, `.hero__tagline` |
| About | `.about__text p` (2 paragraphs), skill tags |
| Experience | `.timeline__role`, `.timeline__list li` (all bullet points) |
| Skills | `.skill-category__title` (5 categories) |
| Education | `.education__title`, `.education__institution` (2 items) |
| Certifications | `.certification__title`, `.certification__institution` |
| Projects | `.project-card__desc` (4 cards) |
| Contact | `.section__title`, `.section__subtitle`, form labels, placeholders, button text, status messages |
| Footer | copyright text |

### Skill Tags

Skill tags (C#, .NET, AWS, etc.) are proper nouns — they stay the same across languages. No translation needed.

## TypeScript Module

```ts
// ts/i18n.ts

type Lang = 'es' | 'en';

const STORAGE_KEY = 'lang';
const DEFAULT_LANG: Lang = 'es';

function getInitialLang(): Lang {
    const saved = localStorage.getItem(STORAGE_KEY);
    if (saved === 'es' || saved === 'en') return saved;
    const browserLang = navigator.language.slice(0, 2);
    return browserLang === 'en' ? 'en' : DEFAULT_LANG;
}

function applyLang(lang: Lang): void {
    document.documentElement.setAttribute('lang', lang);
    document.querySelectorAll<HTMLElement>('[data-es]').forEach(el => {
        const text = el.getAttribute(`data-${lang}`);
        if (text) el.textContent = text;
    });
    // Update toggle label
    const label = document.querySelector('.lang-toggle__label');
    if (label) label.textContent = lang.toUpperCase();
}

function toggleLang(): void {
    const current = document.documentElement.getAttribute('lang') as Lang;
    const next: Lang = current === 'es' ? 'en' : 'es';
    localStorage.setItem(STORAGE_KEY, next);
    applyLang(next);
}

// Initialize
applyLang(getInitialLang());
document.getElementById('lang-toggle')?.addEventListener('click', toggleLang);
```

## SEO Considerations

- `<html lang="es">` / `<html lang="en">` updated dynamically
- Add `<link rel="alternate" hreflang="es" href="https://agigena.com/">` and `<link rel="alternate" hreflang="en" href="https://agigena.com/">` to `<head>`
- Default content in HTML is Spanish (primary audience)

## Testing

1. Load page → should show Spanish by default
2. Click toggle → all text switches to English
3. Click toggle again → back to Spanish
4. Refresh → stays in selected language (localStorage)
5. New visitor with English browser → shows English automatically
6. Check that skill tags (C#, .NET, etc.) remain unchanged
